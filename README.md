# AngryBirdsC - C 语言 & Raylib 实现的经典物理益智游戏

本工程是基于 **C99 标准** 与 **Raylib** 游戏引擎构建的物理益智类游戏（仿《愤怒的小鸟》）。项目包含主菜单系统、Material Design 3 风格 UI、25 个关卡、独立的 2D 冲量物理引擎、特种小鸟技能以及可视化关卡编辑器。

---

## 🚀 技术栈 (Technology Stack)

- **核心语言**：C99标准，保证极佳的性能、极低的内存开销与跨平台兼容性。

- **图形与窗口系统**：Raylib

  处理窗口管理、图形渲染、原生 TTF 字体导入及事件响应。

  使用 `RenderTexture2D` 渲染至 `1200x800` 虚拟分辨率，支持窗口动态缩放时的自动双线性过滤与黑边适配（Letterbox/Pillarbox）。

- **物理与碰撞解算**：自主开发的 2D 冲量物理引擎，针对多层城堡堆叠的稳定性专门优化。支持多子步迭代（Sub-stepping）、相对冲量伤害计算、静态阻尼和对齐硬锁定。
- **数据解析与存档**：cJSON，解析 JSON 格式的关卡数据，并自动读写玩家关卡进度存档。

## 🎨 核心功能与亮点 (Key Features)

1. **25 个递进关卡**：关卡难度从基础的单层城堡逐渐过渡到复杂的多塔、高塔和多鸟大决战。
2. **动态材质系统**：
    *   **石材 (Stone)**：质量 10.0f，摩擦力较高，最稳固，需要高强度撞击才能摧毁。
    *   **木材 (Wood)**：质量 7.0f，易损坏，属性均衡。
    *   **玻璃 (Glass)**：质量 5.0f，摩擦力较低，极易碎。
3. **四大特种小鸟与技能**：
    *   **红鸟 (Red)**：基础款，属性均衡。
    *   **蓝鸟 (Blue)**：飞行中点击可分裂为三只小鸟（原方向 $\pm 15^\circ$ 散射），对玻璃材质有 **6 倍伤害加成**。
    *   **黄鸟 (Yellow)**：飞行中点击可获得 **2.5 倍冲刺加速**，适合击穿木质结构。
    *   **黑鸟 (Black)**：飞行中点击或撞击后爆炸，产生半径 `150px` 的爆炸场，施加最高 `600` 点物理冲击。
4. **Material Design 3 风格 UI**：
    *   基于 MD3 调色板设计，包含阴影层叠（Elevation Shadow）、圆角卡片以及支持缩放动画的交互按钮。
    *   利用 Raylib 导入 `simhei.ttf` 原生汉字字形，支持中文本地化文本渲染。

## 🛠 物理引擎与架构设计

### 堆叠稳定性优化 (`src/entities.c` / `src/physics.c`)

为了保证 2D 刚体在多层高塔堆叠下不发生“滑移”或“抖动”，物理引擎引入了以下机制：

1. **八子步物理更新 (8-Sub-steps)**：将每帧的物理更新细分为 8 次微时间片（`dt / 8`），防止高速小鸟穿模。
2. **角度与速度锁死 (Snapping & Damping)**：
    *   当结构块的线速度低于 `8.0f/12.0f` 且角速度低于 `0.35f` 时，施加额外阻尼，加速其静止。
    *   限制最大偏角偏差在 `0.22` 弧度（约 $12.6^\circ$）内。
    *   偏角在 `0.087` 弧度（约 $5^\circ$）内时，直接硬锁定为直角，确保城堡对齐稳定。
3. **位置纠偏 (Stabilization Bias)**：
    *   计算穿透纠正速度：`bias = 60.0f * (penetration - 0.02f)`。
    *   限制最大纠偏上限（`40.0f`），避免碰撞修正过猛导致刚体飞弹。
4. **双重破坏阈值**：
    *   结构内部沉降的撞击破坏阈值设为较高的 `280.0f`，防止高层塔结构因自重挤压而在初始化时自动垮塌。
    *   小鸟撞击或爆炸冲击的常规破坏阈值设为 `40.0f`。
5. **质量比例调优**：调整石/木/玻璃的质量比为 `10.0 : 7.0 : 5.0`（最大 2 倍差异），避免了物理引擎在计算极端质量差时产生的数值不稳定。

### 关卡程序化生成 (`src/level.c`)

*   关卡在 `assets/levels/*.json` 中定义。
*   `BuildCastle` 函数根据 JSON 中提供的木/石/玻璃材料数量配额，程序化生成多层城堡，并将猪只生成在合理的位置。非顶层天花板会自动避免使用易碎的玻璃块。

## 📂 目录结构

```text
AngryBirdsC/
├── CMakeLists.txt              # CMake 构建配置文件
├── assets/                     # 美术、字体与关卡资源
│   ├── simhei.ttf              # 汉字字体包
│   ├── progress.json           # 玩家关卡进度存档 (自动读写)
│   └── levels/                 # 25 个关卡 JSON 配置 (level1 - level25)
├── src/                        # 源代码
│   ├── main.c                  # 游戏流程控制、状态机、视口管理与技能逻辑
│   ├── entities.h / .c         # 刚体创建、冲量解算、阻尼与对齐限制
│   ├── physics.h / .c          # 物理迭代调度与爆炸场模型
│   ├── slingshot.h / .c        # 弹弓控制与轨迹线预测
│   ├── level.h / .c            # 关卡读取与城堡程序化生成
│   ├── ui.h / .c               # 按钮、卡片、阴影及中文字体初始化
│   └── editor.h / .c           # 关卡编辑器实现
└── build/                      # 编译生成目录
```

## 🏗 编译与运行

项目使用标准的 **CMake** 构建。在首次编译配置时，CMake 会自动下载并链接 **Raylib** 与 **cJSON** 的官方源码进行静态编译，不需要手动配置依赖。

### 📋 编译要求
*   **CMake** (3.14 及以上)
*   **C 编译器**：支持 C99（如 GCC, Clang, MSVC 等）

### 💻 编译命令

1.  打开终端并定位到项目根目录。
2.  创建并进入构建目录：
    ```bash
    mkdir build
    cd build
    ```
3.  生成构建工程：
    ```bash
    cmake ..
    ```
4.  执行编译构建：
    ```bash
    cmake --build . --config Release
    ```

### 📦 自动化打包

项目配置了自动打包命令，运行后会将最新构建的可执行文件和 `assets` 资源目录汇总至 `bird-release` 文件夹下，并压缩打包为 `bird-release.zip`（输出于项目根目录下）：

```bash
cmake --build . --config Release --target package_release
```

### 🎮 运行游戏

编译成功后，相关的可执行文件和 `assets` 资源会被自动复制到 build 目录下。

*   **Windows**：运行 `build/bird-release.exe`。
*   **Linux / macOS**：运行 `build/bird-release`。

## ⌨️ 游戏操作

| 操作 | 作用场景 | 具体功能描述 |
| :--- | :--- | :--- |
| **鼠标左键 (拖拽)** | 游戏内 | 拖拽弹弓上的小鸟，调整发射力度与角度，松开即发射 |
| **鼠标左键 (点击)** | 小鸟飞出后 | 小鸟落地前，点击屏幕任意位置释放特种技能（加速/分裂/爆炸） |
| **TAB 键** | 游戏内 | 开启/关闭可视化关卡编辑器 |
| **鼠标左键 (点击)** | 编辑器开启时 | 在鼠标当前位置生成选中类型的结构块或猪 |
| **回车键 (Enter)** | 游戏主菜单 | 快速进入选关界面 |
| **ESC 键** | 选关界面 | 返回主菜单 |
| **UI 按钮 (左上角)** | 游戏内 | 点击重试当前关卡，或者返回选关界面 |
