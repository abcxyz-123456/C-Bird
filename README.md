# AngryBirdsC - 经典物理益智游戏 (C语言 & Raylib 实现)

本开发项目是一个基于 **C99 标准** 与 **Raylib** 游戏引擎构建的高品质、高可玩性的物理益智类游戏（仿《愤怒的小鸟》）。项目包含完整的主菜单系统、扁平化 Material Design 3 风格 UI、25个精心设计的关卡、独立的物理模拟核心、特种小鸟能力以及内置的可视化关卡编辑器。

---

## 🚀 技术栈 (Technology Stack)

项目坚持“极简依赖、高效原生”的开发哲学，技术选型如下：

1. **核心语言**：**C99 标准** (C Language)
   - 提供极佳的性能、极低的内存开销与跨平台编译兼容性。
2. **图形与窗口系统**：**Raylib** (GitHub: `raysan5/raylib`)
   - 负责图形渲染、视口管理、原生字体导入与事件响应。
   - 使用 **RenderTexture2D** 实现 `1200x800` 虚拟分辨率渲染，并支持动态窗口缩放时的自动**双线性过滤 (Bilinear Filtering)** 与 **黑边适配模式 (Letterbox/Pillarbox)**。
3. **物理与碰撞解算器**：**自主研发的 2D 冲量物理引擎**
   - 针对城堡重力堆叠稳定性而定制。包含多子步解算（Sub-stepping）、相对冲量伤害模型、静止角度校正与防抖锁定。
   - *(注：虽然 `CMakeLists.txt` 中配置了 Chipmunk2D 依赖作为预留，但为了保证重力建筑堆叠的完美稳定性，当前版本采用深度定制的 C 冲量物理引擎)*
4. **数据解析与存档**：**cJSON** (GitHub: `DaveGamble/cJSON`)
   - 用于关卡数据格式（JSON）解析以及玩家关卡进度（星级、解锁状态）的存档读写。

## 🎨 核心功能与亮点 (Key Features)

- **25 个渐进式关卡**：从简单的单层木盒挑战，到庞大石墙、多重哨所以及终极的 16 只小鸟世纪决战，完美覆盖新手教学与硬核解密。
- **动态材质与损毁系统**：
  - **木材 (Wood)**：中等质量与阻尼，易损坏。
  - **石材 (Stone)**：高质量与高摩擦力，非常沉重牢固，但受到重击时会产生裂纹。
  - **玻璃 (Glass)**：轻质、低摩擦力，极其脆弱。
- **四大特种小鸟与飞行动能**：
  - **红鸟 (Red)**：标准先锋，各项属性均衡。
  - **蓝鸟 (Blue)**：轻巧迅捷。飞行中点击屏幕分裂为三只小鸟（在原速度方向上产生 $\pm 15^\circ$ 的旋转散射），针对玻璃材质拥有 **6 倍伤害加成**。
  - **黄鸟 (Yellow)**：穿透专家。飞行中点击屏幕可获得 **2.5 倍超速爆发冲锋**，刺穿重重障碍。
  - **黑鸟 (Black)**：爆破艺术家。飞行中点击屏幕直接引爆，对半径 `150px` 内的所有刚体施加最高 `600` 点爆炸冲击力，化顽石为齑粉。
- **Material Design 3 风格 UI**：
  - 基于 MD3 Tonal Palette 调色板设计，呈现高阶审美。
  - 拥有平滑的阴影层叠（Elevation Shadow）、圆角卡片、带有动态缩放和阻尼过渡的按钮。
  - 支持原生 **TTF 汉字字型加载 (基于 SimHei)**，实现无瑕疵的中文本地化文本排版。
- **内置关卡编辑器**：
  - 关卡运行中随时按 `TAB` 键切入/切出可视化关卡编辑器。
  - 支持鼠标悬停点击摆放刚体、猪只，为自定义关卡开发与关卡物理调试提供了极大便利。

## 🛠 架构设计与实现方案 (Architecture & Core Systems)

### 1. 极简冲量解算物理引擎 (`entities.c` / `physics.c`)
为了解决 2D 刚体在静态堆叠下容易产生“震颤”、“滑移”甚至“自爆”的行业难题，物理引擎引入了多层优化：
- **八子步物理迭代 (8-Sub-steps)**：将单帧物理更新细分为 8 次超小时间片更新（`dt / 8`），极大地避免了高速运动物体（如飞行小鸟）的“穿模”或“瞬移”问题。
- **角度限制与静止锁死 (Wobble & Snapping Limit)**：
  - 当检测到结构块（Block）处于极低速度与低角速度状态时，将其阻尼迅速提高，防止无限微小震荡。
  - 将块体倾角限制在与最近的 $90^\circ$ 状态相差不超过 `0.22` 弧度（约 $12.6^\circ$）。
  - 一旦夹角微调至 `0.087` 弧度（约 $5^\circ$）以内，系统会将块体角度**瞬间硬锁定**到 $90^\circ$ 增量，这让直角堆叠城堡固若金汤。
- **高阶位置纠偏 (Stabilization Bias)**：
  - 采用穿透深度反馈纠偏：`bias = 120.0f * (penetration - 0.02f)`。
  - 设置最大纠偏速度上限（`150.0f`），避免积压的刚体发生“火山爆发”式的弹射。
- **双重冲量伤害算法**：
  - **结构静态沉降防损**：块体之间的堆叠沉降使用极高的破坏阈值（`280.0f`），使下方承重层免受由于重力挤压而自我解体的威胁。
  - **外部飞击损毁**：由小鸟碰撞及爆炸产生的真正动能碰撞使用较低的破坏阈值（`40.0f`），并基于物理受力计算生命扣除。

### 2. 场景的程序化配置与生成 (`level.c`)
- 每个关卡都使用轻量 JSON 文件独立描述（放置在 `assets/levels/` 中）。
- JSON 配置定义了本关的小鸟配置队列表（不同小鸟的组合）、猪只数量、城堡数量以及结构块材料库存（wood/stone/glass的数量）。
- **`BuildCastle` 函数**在关卡初始化时执行。它采用**材质池配额抽卡机制**，基于材料库存分配建筑的柱、梁，并加入防止承重层出现脆弱玻璃块的**建筑力学合理性校验算法**，程序化生成高耸且稳定的物理城堡。

### 3. 多比例适配的视口系统 (`main.c`)
- 通过 `BeginTextureMode` 将游戏画面统一渲染至 `1200x800` 虚拟纹理中。
- 在 `BeginDrawing` 阶段，通过对比屏幕宽高比自动计算出最佳的拉伸缩放系数（`scale`）与居中偏移量（`offset`），对纹理进行上下/左右黑边填充拉伸，并运用双线性过滤保持图像边缘平滑。
- 输入端通过 `Gui_SetMousePosition` 自动完成物理坐标与虚拟视口坐标的反向转换，保证了缩放模式下拖拽弹弓与点击技能的精度丝毫不差。

## 🏗 构建与编译指南 (Build & Run Guide)

项目完全基于标准 **CMake** 组织构建。得益于 `FetchContent` 模块，项目无需手动安装任何第三方库，在配置编译时会自动联网下载 **Raylib**、**Chipmunk2D** 与 **cJSON** 的官方源码并进行本地静态编译链接。

### 📋 构建前置要求
- **CMake** (3.14 及以上版本)
- 支持 C99 的 C 编译器：
  - **Windows**：MinGW (GCC) 或 Visual Studio (MSVC)
  - **macOS**：Clang / Xcode Command Line Tools
  - **Linux**：GCC

---

### 💻 编译步骤

#### 1. 克隆或解压项目到本地目录
确保终端或命令提示符（CMD/PowerShell）已定位到项目根目录（例如 `d:\programs\others\bird`）。

#### 2. 创建并进入构建目录
```bash
mkdir build
cd build
```

#### 3. 配置项目并生成 Makefile / 工程文件
- **对于 Windows (使用 MinGW / Makefiles)**：
  ```bash
  cmake -G "MinGW Makefiles" ..
  ```
- **对于 Windows (使用 Visual Studio)**：
  ```bash
  cmake ..
  ```
- **对于 macOS / Linux**：
  ```bash
  cmake ..
  ```

#### 4. 执行编译构建
使用 CMake 的跨平台编译指令（支持自动配置 Release 模式）：
```bash
cmake --build . --config Release
```

*提示：编译过程中，CMake 会拉取 Raylib 和 cJSON 的 Git 仓库源码并进行本地编译，首次构建可能需要 1~2 分钟，具体取决于您的网络状况。*

---

### 🎮 运行游戏

编译成功后，可执行程序和 `assets` 资源文件夹会自动部署到编译目录中。

- **Windows**：双击运行 `build/AngryBirdsC.exe` 或在终端运行 `./AngryBirdsC`。
- **macOS / Linux**：在终端运行 `./AngryBirdsC`。
